Infinidi Gaming Platform MVP

Conducted user research with 1,000+ gamers to develop the Infinidi gaming platform MVP at Affinidi, defining product scope, test cases, and sprint workflow for the gaming ecosystem.

Organization Affinidi
Role Product Manager
Key Achievement 1,000+ Users Researched
Key Tools Figma, Notion, Jira, UX Research

Overview

Infinidi is a gaming platform that connects gamers, game developers, and gaming communities in a decentralized ecosystem. As Product Manager, I led the MVP development from concept to launch, conducting extensive user research with over 1,000 gamers to ensure we built a product that truly addressed user needs.

This project showcased my ability to combine user research, product strategy, and technical execution to build products that resonate with users. The MVP successfully validated core product hypotheses and established a foundation for future platform growth.

Challenge

Building a gaming platform MVP presented several challenges:

  • Understanding User Needs: The gaming ecosystem is diverse—we needed to understand what different segments of gamers actually wanted
  • Defining MVP Scope: With limited resources, we needed to identify the minimum set of features that would deliver value
  • Technical Complexity: Building a platform that connects multiple stakeholders (gamers, developers, communities) required careful architecture decisions
  • Competitive Landscape: The gaming platform space is crowded—we needed to differentiate through user experience and unique value propositions
  • Rapid Iteration: Gaming is a fast-moving industry—we needed to move quickly while maintaining quality

Approach

1. Comprehensive User Research

I conducted extensive user research to understand the gaming ecosystem:

  • 1,000+ User Interviews: Conducted structured interviews with gamers across different segments (casual, competitive, indie developers, streamers)
  • Survey Analysis: Distributed surveys to gather quantitative data on gaming habits, platform preferences, and pain points
  • Competitive Analysis: Analyzed existing gaming platforms to identify gaps and opportunities
  • User Personas: Created detailed personas based on research findings to guide product decisions
  • Journey Mapping: Mapped user journeys to identify friction points and opportunities for improvement

This research revealed key insights:

  • Gamers wanted better ways to discover indie games and connect with developers
  • Game developers needed more direct channels to reach their audience
  • Gaming communities wanted tools to organize events and tournaments
  • There was demand for a platform that rewarded both creators and players

2. MVP Scope Definition

Based on user research, I defined the MVP scope using a structured prioritization framework:

  • Core Features: User profiles, game discovery, basic community features
  • Must-Have Integrations: Authentication, payment processing, game library connections
  • Nice-to-Have (Post-MVP): Advanced tournament features, streaming integration, advanced analytics

I created detailed product requirements documents (PRDs) in Notion, including user stories, acceptance criteria, and technical specifications.

3. Test Case Development

I worked closely with QA engineers to develop comprehensive test cases:

  • Functional Testing: Test cases for all core user flows (signup, game discovery, community creation)
  • Integration Testing: Test cases for third-party integrations (payment, authentication, game APIs)
  • Performance Testing: Test cases to ensure platform could handle expected user load
  • User Acceptance Testing: Test cases based on user research findings to validate product meets user needs

This rigorous testing approach ensured the MVP was stable and met user expectations at launch.

4. Sprint Workflow & Agile Delivery

I established and facilitated Agile workflows:

  • Sprint Planning: 2-week sprints with clear goals and deliverables
  • Daily Standups: Ensured team alignment and quick issue resolution
  • Sprint Reviews: Demoed progress to stakeholders and gathered feedback
  • Retrospectives: Continuously improved team processes and velocity
  • Backlog Management: Prioritized features based on user research insights and business value

I used Jira to track all user stories, bugs, and tasks, ensuring visibility across the team.

5. Design & UX

I collaborated with designers to create user-centered designs:

  • Created wireframes in Figma based on user research findings
  • Conducted usability testing with target users
  • Iterated on designs based on feedback
  • Ensured designs were accessible and responsive

The MVP launch delivered significant value:

  • Validated Product-Market Fit: User research and MVP launch validated that there was demand for the platform
  • Established Foundation: MVP provided a solid foundation for future feature development
  • User Engagement: Early users provided valuable feedback that informed product roadmap
  • Team Alignment: Clear scope and test cases ensured all team members were aligned on goals

Key Learnings

This project reinforced important lessons about product management:

  • User Research is Essential: Conducting research with 1,000+ users provided insights that directly informed product decisions and prevented building the wrong features
  • MVP Scope is Critical: Defining clear MVP scope prevents feature creep and ensures focus on what matters most
  • Test Cases Guide Quality: Comprehensive test cases ensure product quality and user satisfaction
  • Agile Enables Speed: Well-facilitated Agile workflows enable rapid iteration while maintaining quality
  • Cross-Functional Collaboration: Success required close collaboration with designers, engineers, and QA

Team Structure: Cross-functional team including product, design, engineering, and QA
